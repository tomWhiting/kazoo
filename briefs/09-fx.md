# FX — Effects Rack

## What It Is

A modular effects processor. Provides reverb, delay, chorus, distortion, EQ, and compression as standalone units that can be chained in any order. Instruments can use these via send/return through Mixbus, or the FX crate can process a WAV file directly.

## What It Does

### Effects Units

Each effect is a self-contained processor with its own parameter set. They share a common trait interface so they can be chained arbitrarily.

#### Reverb (Algorithmic)

Schroeder-Moorer style: 4 parallel comb filters fed into 2 allpass filters in series.

| Parameter | Range | Description |
|-----------|-------|-------------|
| **room_size** | 0.0 - 1.0 | Comb filter feedback. Larger = longer tail. |
| **damping** | 0.0 - 1.0 | High-frequency absorption. Higher = darker reverb. |
| **width** | 0.0 - 1.0 | Stereo spread. 0 = mono, 1 = full stereo. |
| **mix** | 0.0 - 1.0 | Dry/wet blend. |
| **pre_delay_ms** | 0 - 100 | Delay before reverb onset. Adds space. |

#### Delay (Tempo-Synced)

Stereo ping-pong delay with optional sync to Pulse clock.

| Parameter | Range | Description |
|-----------|-------|-------------|
| **time_ms** | 1 - 2000 | Delay time (ignored if synced). |
| **sync_rate** | Option | "1/4", "1/8", "1/8T", "1/16", etc. Overrides time_ms. |
| **feedback** | 0.0 - 0.95 | Repeat amount. Capped below 1.0 to prevent runaway. |
| **ping_pong** | bool | Alternate repeats left/right. |
| **hp_cutoff** | 20 - 20000 | High-pass on feedback path. Thins repeats. |
| **lp_cutoff** | 20 - 20000 | Low-pass on feedback path. Darkens repeats. |
| **mix** | 0.0 - 1.0 | Dry/wet blend. |

#### Chorus

Stereo chorus using modulated delay lines.

| Parameter | Range | Description |
|-----------|-------|-------------|
| **rate** | 0.1 - 10.0 Hz | LFO modulation speed. |
| **depth** | 0.0 - 1.0 | Modulation amount (delay excursion). |
| **voices** | 1 - 4 | Number of chorus voices (more = thicker). |
| **mix** | 0.0 - 1.0 | Dry/wet blend. |

#### Distortion

Multiple distortion algorithms selectable by type.

| Type | Description |
|------|-------------|
| **soft_clip** | Tanh waveshaping. Warm, tube-like. |
| **hard_clip** | Flat ceiling clip. Harsh, digital. |
| **foldback** | Signal folds back at threshold. Weird, glitchy harmonics. |
| **bitcrush** | Reduce bit depth and/or sample rate. Lo-fi. |

| Parameter | Range | Description |
|-----------|-------|-------------|
| **type** | enum | Distortion algorithm. |
| **drive** | 0.0 - 1.0 | Input gain before shaping. |
| **tone** | 0.0 - 1.0 | Post-distortion low-pass. 0 = dark, 1 = bright. |
| **bit_depth** | 2 - 16 | For bitcrush mode. |
| **sample_rate_divisor** | 1 - 64 | For bitcrush mode. |
| **mix** | 0.0 - 1.0 | Dry/wet blend. |

#### EQ (Parametric)

3-band parametric EQ with low shelf, mid peak, high shelf.

| Parameter | Range | Description |
|-----------|-------|-------------|
| **low_freq** | 20 - 500 Hz | Low shelf frequency. |
| **low_gain_db** | -12 - +12 | Low shelf gain. |
| **mid_freq** | 200 - 8000 Hz | Mid peak frequency. |
| **mid_gain_db** | -12 - +12 | Mid peak gain. |
| **mid_q** | 0.1 - 10.0 | Mid peak bandwidth. |
| **high_freq** | 2000 - 20000 Hz | High shelf frequency. |
| **high_gain_db** | -12 - +12 | High shelf gain. |

#### Compressor

Feed-forward compressor with sidechain capability.

| Parameter | Range | Description |
|-----------|-------|-------------|
| **threshold_db** | -60 - 0 | Level where compression starts. |
| **ratio** | 1.0 - 20.0 | Compression ratio. 20:1 ≈ limiting. |
| **attack_ms** | 0.1 - 100 | How fast gain reduction kicks in. |
| **release_ms** | 10 - 1000 | How fast gain reduction releases. |
| **makeup_gain_db** | 0 - 24 | Post-compression gain boost. |
| **knee_db** | 0 - 12 | Soft knee width. 0 = hard knee. |

### Effect Chain

Effects are chained in order. Audio flows through each effect sequentially:

```
Input → [Effect 1] → [Effect 2] → ... → [Effect N] → Output
```

The chain is defined as a JSON array. Order matters — distortion before reverb sounds very different from reverb before distortion.

### Standalone Processing

The FX crate can load a WAV file, process it through an effect chain, and write the result. Useful for post-processing instrument renders.

## JSON Input

```json
{
  "input": "dry_signal.wav",
  "chain": [
    {
      "type": "eq",
      "low_freq": 80,
      "low_gain_db": 3,
      "mid_freq": 2000,
      "mid_gain_db": -2,
      "mid_q": 1.5,
      "high_freq": 10000,
      "high_gain_db": 1
    },
    {
      "type": "compressor",
      "threshold_db": -18,
      "ratio": 4.0,
      "attack_ms": 10,
      "release_ms": 100,
      "makeup_gain_db": 6,
      "knee_db": 3
    },
    {
      "type": "delay",
      "sync_rate": "1/8",
      "feedback": 0.4,
      "ping_pong": true,
      "lp_cutoff": 4000,
      "mix": 0.25
    },
    {
      "type": "reverb",
      "room_size": 0.7,
      "damping": 0.5,
      "width": 1.0,
      "pre_delay_ms": 20,
      "mix": 0.3
    }
  ],
  "output": "processed_signal.wav",
  "bpm": 120
}
```

BPM is optional — only needed if any effect uses tempo sync.

## Integration with Mixbus

Mixbus defines send/return buses. Each send bus routes to an FX chain instance. The return audio is mixed back into the master bus.

```
Channel (snare) ──[send 0.4]──→ Reverb FX Chain ──→ Return to Master
Channel (lead)  ──[send 0.6]──→ Reverb FX Chain ──→ Return to Master
Channel (bass)  ──[send 0.0]──→ (no reverb on bass)
```

Multiple send buses possible (e.g., reverb bus + delay bus).

## Trait Interface

```rust
pub trait Effect: Send {
    fn process_stereo(&mut self, input: &mut [[f32; 2]]);
    fn set_sample_rate(&mut self, sample_rate: u32);
    fn reset(&mut self);  // Clear delay lines, envelopes, etc.
    fn tail_samples(&self) -> usize;  // How many samples of tail after input stops
}
```

All effects implement this trait. The chain is `Vec<Box<dyn Effect>>`.

## Dependencies

- `dsp` crate (filters, delay lines, oscillators for LFO modulation)
- `pulse` crate (optional, for tempo-synced delay/chorus)
- `serde` + `serde_json`
- `hound` (WAV I/O)

## Testing

- Reverb: process an impulse (single sample spike), verify decaying tail in output.
- Delay: process a click at 120 BPM with 1/4 sync, verify repeats at exact beat intervals.
- Chorus: process a sine wave, verify output has frequency spread (detuned copies).
- Distortion: process a sine with drive, verify harmonics added. Bitcrush: verify stepped waveform.
- EQ: boost 1kHz by 12dB, verify 1kHz amplitude increase in FFT.
- Compressor: feed signal that crosses threshold, verify gain reduction and makeup.
- Chain order: compare distortion→reverb vs reverb→distortion, verify different output.
- Standalone: load a WAV, process through chain, write output, verify non-silent and correct duration.
